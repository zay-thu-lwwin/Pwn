	တကယ့်ကွန်ပျူတာ အစစ်မဟုတ်
	ကွန်ပျူတာအမျိုးမျိုးကြောင့် "ဒီ Algorithm က တကယ်ပဲ မြန်သလား? နှေးသလား?" ဆိုတာ တိုင်းတာဖို့ ခက်ခဲသွား
	ဒီစံပြ ကွန်ပျူတာပေါ်မှာ ပြေးတဲ့အခါမှ Algorithm တစ်ခုရဲ့ မြန်နှုန်း (Time Complexity) ကို အတိအကျ တိုင်းတာလို့ရ
	Theoretical Model
RAM (Random Access Machine) တစ်ခုမှာ အဓိက အစိတ်အပိုင်း ၃ ခု ရှိ

	- Input Tape: ကွန်ပျူတာထဲကို ထည့်ပေးလိုက်တဲ့ ကိန်းဂဏန်းတွေ (Data) ရှိတဲ့နေရာ။ (Read-only - ဖတ်လို့ပဲ ရမယ်)။
	- Output Tape: အဖြေထွက်လာမယ့်နေရာ။ (Write-only - ရေးလို့ပဲ ရမယ်)။
	- Memory (Registers): ဒါက အရေးကြီးဆုံးပါ။ r0​,r1​,r2​,… စသဖြင့် အကန့်အသတ်မရှိတဲ့ Register အကွက်လေးတွေ (Special Register (r0​): ဒါကို Accumulator လို့ ခေါ်တယ်။ တွက်ချက်မှု အားလုံး (ပေါင်း၊ နှုတ်၊ မြှောက်၊ စား) ကို ဒီ r0​ ထဲမှာပဲ လုပ်ရ)


	RAM Program တစ်ခု ရေးဖို့ အသုံးအများဆုံး Command ၆ ခု

| **Command** | **အဓိပ္ပာယ်**   | **ရှင်းလင်းချက်**                                                           |
| ----------- | --------------- | --------------------------------------------------------------------------- |
| `READ`      | Input မှ ဖတ်ရန် | Input Tape က ဂဏန်းကိုယူပြီး $r_0$ ထဲ ထည့်လိုက်တာပါ။                         |
| `STORE i`   | သိမ်းဆည်းရန်    | $r_0$ ထဲက ဂဏန်းကို Register $i$ ($r_i$) ထဲ သွားသိမ်းတာပါ။                   |
| `LOAD i`    | ပြန်ခေါ်ရန်     | Register $i$ ($r_i$) ထဲက ဂဏန်းကို $r_0$ ထဲ ပြန်ယူလာတာပါ။                    |
| `ADD i`     | ပေါင်းရန်       | $r_0$ ထဲက ဂဏန်းနဲ့ $r_i$ ထဲက ဂဏန်းကို ပေါင်းပြီး $r_0$ ထဲမှာပဲ အဖြေထားတာပါ။ |
| `WRITE`     | ထုတ်ရန်         | $r_0$ ထဲက အဖြေကို Output Tape ပေါ် တင်လိုက်တာပါ။                            |
| `HALT`      | ရပ်ရန်          | Program ကို အဆုံးသတ်လိုက်တာပါ။                                              |



Logarithmic cost model 

logarithmic က အရွယ်အစား ( computer မှာဆိုတော့ bits တွေ) ကို ထည့်စဉ်းစားတယ် 
အဲကောင်ရဲ့ time complexity မှာဆို
ကိန်းဂဏန်း ကြီးရင်ကြီးသလောက် ကြာမယ် (2+2 က တွက်ရင် မကြာပေမယ့် 5435354353+253455453 ဆို တွက်ရင် ကြာမယ့်သဘော)
အဲကောင်ရဲ့ space complexity မှာက
register ဘယ်နှစ်ခုယူလဲဆိုတာ ပြတယ် ဒါမဲ့ ကိန်းဂဏန်း ကြီးရင်ကြီးသလောက် register size ကလည်းကြီးလာမယ့်သဘော

Uniform cost model 

uniform က အရွယ်အစားကို ထည့်မစဉ်းစား
အဲကောင်နဲ့ time complexity မှာဆို
ဘယ်နှစ်ကြိမ်လဲဆိုတာဘဲစဉ်းစား ကိန်းဂဏန်း ဘယ်လောက်ကြီးလဲဆိုတာထည့်မစဉ်းစား
(2+2 က တွက်ရင်လည်း တစ်ကြိမ်ဘဲ 5435354353+253455453 တွက်ရင် လည်းတစ်ကြိမ်ဘဲ)
အဲကောင်ရဲ့ spaceမှာဆို 
register ဘယ်လောက်ယူမှာလဲဆိုတာစဉ်းစား register size ကိုထည့်မတွက် 

- O(1) ဆိုတာ constant, ဘယ်နှစ်ကြိမ်ဘဲ run run , ဒီအရေအတွက်ဘဲပုံသေ
- O(n) ဆိုတာ အချိုးကျ , n များရင် များလာမယ်ဆိုတဲ့သဘော
- O(n^2) ဆိုတာ အလုပ် ၂ ဆတိုးရင် အချိန် ၄ ဆ ကြာမယ်
