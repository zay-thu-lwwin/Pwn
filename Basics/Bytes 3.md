
#### Bit Movement Operations


##### Left Shift (<<)

bits တွေကို ဘယ်ဘက်ရွှေ့တာ 
ဆိုတော့ ဘယ်ဘက်စွန်းက 0 ကိုဖျောက်ပြီး
	ညာဘက်စွန်းကို 0 ထည့် (ညာဘက်က 0 အမြဲထည့်/  စက်တစ်ခုပေါ်မှာရွေ့သွားတယ်လို့မျက်လုံးထဲမြင်ကြည့်)
ဒီနေရာမှာ bits တန်းဖိုးပြောင်းသွာတယ်
0တစ်လုံးနေရာစာအတွက် x2 ဆ ယူတယ်

```
ဥပမာ - 8-bit: 0b00001010 (10) ကို ဘယ်ဘက် 2 နေရာရွှေ့

အစ:    0 0 0 0 1 0 1 0
           ↓ ← shift 2 ←
အဆင့် 1: 0 0 0 1 0 1 0 _
အဆင့် 2: 0 0 1 0 1 0 _ _
နောက်ဆုံး: 0 0 1 0 1 0 [0 0] = 0b00101000 (40) 

ဒီမှာ 0 2နေရာစာအတွက်ဆိုတော့ x4(4ဆ)ဖြစ်မယ်

```

တစ်ခုသတိထားရမှာက ဘယ်ဘက်အစွန်းက 1 ဆိုရင်  x2(2ဆ)တွက်လို့မရတော့ဘူး
တန်ဖိုးကလျော့သွားတယ်
overflow ဖြစ်တာလို့ခေါ်တယ်

```
   1 0 1 1   (11)
<< 1

ဘယ်အစွန်းဆုံး 1 ဖျက်ခံရမယ်
ညာဘက်က လွတ်နေတဲ့နေရာထဲ 0 ထည့်မယ်

= 0 1 1 0   (6)
```

left shift မှာဆို (-) ကနေ (+) ဖြစ်နိုင်
```c
+3 = 0011 → <<1 → 0110 = +6 ✅
-5 = 1011 → <<1 → 0110 = +6 ❌ (အနုတ်က အပေါင်းဖြစ်သွား)
```


##### Right Shift (>>) (for unsigned)

အပေါ်က ကောင်နဲ့ပြောင်းပြန်
ပြီးရင် x2(2ဆတိုးတာ မဟုတ်တော့ဘူး ) ÷2 (2ဆလျော့တာဖြစ်သွားပြီ)
ပြီးတော့ စားလဒ်ကိုဘဲယူတယ် အကြွင်းပျောက်တယ်
ဒီမှာ ပျောက်မယ့်ကောင်က 0 ဖြစ်ဖြစ် 1 ဖြစ်ဖြစ် overflow မဖြစ် 2နဲ့စားပြီး စားလဒ်ဘဲယူ
```
   0 0 0 1 0 1 1 0   (22)
>> 1

ညာဘက်အစွန်းဆုံး 0 ဖျက်ခံရတယ်
ဘယ်ဘက်က 0 ထည့်တယ်

= 0 0 0 0 1 0 1 1   (11)   ✅ 22÷2 = 11
```


##### Right shift (Arithmetic) (for signed)

(-)value တွေအတွက်သုံးလို့ရအောင်လို့
ဘာလို့ဆို ငါတို့ က စားတဲ့အခါ စားလဒ်ကိုဘဲယူတာက . ထွက်အောင်စားပြီး   floor ကိုယူတဲ့သဘောလိုဖြစ်တယ်
5 / 2 = 2.5 , floor = 2 , ဆိုတော့ စားလဒ်က 2
bitsတွေမှာ အနုတ်ကိန်းတွေက ပြောင်းပြန် နောက်ကနေ(bitsကြီးတာကနေ)စလာတဲ့ဆိုသိပြီးပြီ
ဆိုတော့ -5 / 2 = -2.5 , floor = -2 ဆိုတောကြီးဖြစ်နေတာ တကယ်တမ်းက -3 ဖြစ်သင့်တာ
ဆိုတော့ Arithmetic right shift မှာက
အစောကအတိုင်း right shift လုပ်မယ်
ပြီးတော့ ဘယ်ဘက်စွန်းကို 0မထည့်ဘဲ 1ထည့်မယ်


| မူရင်း (dec) | Binary (4-bit) | >> 1   | ရလဒ် (dec) | ရှင်းလင်းချင်း   |
| ------------ | -------------- | ------ | ---------- | ---------------- |
| -1           | `1111`         | `1111` | -1         | floor(-1/2) = -1 |
| -2           | `1110`         | `1111` | -1         | floor(-2/2) = -1 |
| -3           | `1101`         | `1110` | -2         | floor(-3/2) = -2 |
| -4           | `1100`         | `1110` | -2         | floor(-4/2) = -2 |
| -5           | `1011`         | `1101` | -3         | floor(-5/2) = -3 |
| -6           | `1010`         | `1101` | -3         | floor(-6/2) = -3 |
| -7           | `1001`         | `1100` | -4         | floor(-7/2) = -4 |
| -8           | `1000`         | `1100` | -4         | floor(-8/2) = -4 |

positive မှာဆို Logical နဲ့ Arithmetic တူတယ်

- **Logical Right Shift** → ဘယ်ဘက်ကို `0` အမြဲဖြည့် (unsigned အတွက်)
- **Arithmetic Right Shift** → ဘယ်ဘက်ကို sign bit အတိုင်းဖြည့် ( + ဆို 0, - ဆို 1) (ဆိုတော့ singed (+) အတွက်ဆို logical right shift နဲ့တူသွားပြီ)

|မူရင်း (dec)|Binary (4-bit)|>> 1|ရလဒ် (dec)|ရှင်းလင်းချက်|
|---|---|---|---|---|
|5|`0101`|`0010`|2|floor(5/2)=2|
|6|`0110`|`0011`|3|floor(6/2)=3|
|7|`0111`|`0011`|3|floor(7/2)=3|
|1|`0001`|`0000`|0|floor(1/2)=0|
(hardware မှာ တစ်ခုတည်းသော right shift ဆိုတာမရှိဘူး
CPU က သူ့မှာ arithmetic right shift နဲ့ logical right shift ဆိုပြီး သီးခြား instruction နှစ်ခုရှိတယ်

ဥပမာ ARM CPU:

- `ASR` (Arithmetic Shift Right)
- `LSR` (Logical Shift Right)
    

x86 CPU:

- `SAR` (Shift Arithmetic Right)
- `SHR` (Shift Logical Right)

| Language   | `>>`                                                  | `>>>`                                |
| ---------- | ----------------------------------------------------- | ------------------------------------ |
| Java       | Arithmetic right shift (sign bit fill)                | Logical right shift (0 fill)         |
| C / C++    | Implementation-defined (mostly arithmetic for signed) | မရှိ (unsigned သုံးပါ)               |
| Python     | Arithmetic right shift (အမြဲ)                         | မရှိ                                 |
| JavaScript | Arithmetic right shift (sign fill)                    | Logical right shift (0 fill) (>>>)   |
| C#         | Arithmetic right shift (sign fill)                    | Logical right shift (unsigned shift) |


##### Rotation

Shift လုပ်တဲ့အခါ အစွန်းက ဘစ်တွေ ပျောက်သွားတယ်
Rotation လုပ်တဲ့အခါ အစွန်းက ဘစ်တွေ မပျောက်ဘူး 
ပျောက်မယ့်အစား တစ်ဖက်ခြမ်းကို ပြန်ပို့ပေးတယ် (စက်ဝိုင်းပုံ)
Rotation မှာ  bit တစ်ခုမှ မဆုံးရှုံးဘူး နေရာပြောင်းရုံပဲ

| Type                   | ဘယ်လိုလုပ်လဲ                                     | instruction |
| ---------------------- | ------------------------------------------------ | ----------- |
| **Rotate Left (ROL)**  | ဘယ်ဘက်ရွှေ့၊ ဘယ်စွန်းက bitကို ညာဘက်ကို ပြန်ထည့်  | `ROL`       |
| **Rotate Right (ROR)** | ညာဘက်ရွှေ့၊ ညာစွန်းက bit  ကို ဘယ်ဘက်ကို ပြန်ထည့် | `ROR`       |

|မူရင်း|Binary|ROL 1|ROR 1|
|---|---|---|---|
|1|`0001`|`0010` (2)|`1000` (8)|
|3|`0011`|`0110` (6)|`1001` (9)|
|5|`0101`|`1010` (10)|`1010` (10)|
|7|`0111`|`1110` (14)|`1011` (11)|
|8|`1000`|`0001` (1)|`0100` (4)|
|15|`1111`|`1111` (15)|`1111` (15)|
Rotate ကို အကြိမ်ကြိမ် ဆက်လုပ်နိုင်

```
မူရင်း:     1 0 1 1
ROL 2:       1 1 1 0
(ဘယ်စွန်း 1 နဲ့ 0 က ညာဘက်ကို ပြန်သွား)
```
